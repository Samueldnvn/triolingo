### Lessons 8-14 (7 lessons, 175 questions)

Built advanced hashing collision resolution lessons:

**Lesson 8: Linear Probing (25 questions, 20 code)**
- Sequential probing: i, i+1, i+2, ...
- Primary clustering problem
- Load factor < 0.7 for good performance
- O(1) average, O(n) worst case
- Excellent cache locality

**Lesson 9: Quadratic Probing (25 questions, 20 code)**
- Quadratic step: h + i²
- Reduces primary clustering
- Prime table size recommended
- Secondary clustering possible
- Worse cache locality than linear

**Lesson 10: Double Hashing (25 questions, 20 code)**
- Two hash functions: h1 and h2
- Probe: (h1 + i×h2) % size
- h2 must be coprime to table size
- Minimal clustering
- Guaranteed full coverage

**Lesson 11: Separate Chaining (25 questions, 20 code)**
- Each bucket is a linked list
- Load factor can exceed 1
- Easy deletion
- Simple implementation
- Pointer overhead

**Lesson 12: Cuckoo Hashing (25 questions, 20 code)**
- Two hash tables
- Displace existing elements
- O(1) worst-case lookup
- Amortized O(1) insert
- Good cache locality

**Lesson 13: Robin Hood Hashing (25 questions, 20 code)**
- Track PSL (Probe Sequence Length)
- Displace "rich" to "poor" (balance)
- Low variance in probe lengths
- Better search than linear
- Requires backshift on deletion

**Lesson 14: Hopscotch Hashing (25 questions, 20 code)**
- Bounded neighborhoods (H buckets)
- Bitmap tracks occupancy
- O(1) worst-case lookup
- Excellent cache locality
- Good for concurrency
