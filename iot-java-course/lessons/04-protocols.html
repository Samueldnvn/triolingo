IoT Communication Protocols

Enabling Device-to-Device and Device-to-Cloud Communication

Communication protocols are the foundation of IoT connectivity, determining how devices exchange data efficiently and reliably.

KEY FACTORS IN PROTOCOL SELECTION
  • Bandwidth: How much network capacity is available
  • Latency: Required response time
  • Power Consumption: Battery life requirements
  • Reliability: Guaranteed vs. best-effort delivery
  • Scalability: Number of devices supported
  • Security: Encryption and authentication capabilities

MQTT (MESSAGE QUEUING TELEMETRY TRANSPORT)

MQTT is a lightweight publish/subscribe messaging protocol designed for IoT devices with limited bandwidth and power.

MQTT CHARACTERISTICS
  • Architecture: Publish/subscribe with broker mediation
  • Transport: Runs over TCP/IP
  • QoS Levels: 0 (at most once), 1 (at least once), 2 (exactly once)
  • Message Size: Very small header (2 bytes minimum)
  • Topics: Hierarchical addressing (e.g., home/livingroom/temperature)
// Java MQTT client example with Eclipse Paho

import org.eclipse.paho.client.mqttv3.*;

public class MqttPublisher {
  public static void main(String[] args) throws Exception {
    String broker = "tcp://broker.hivemq.com:1883";
    String clientId = "JavaClient";

    MqttClient client = new MqttClient(broker, clientId);
    MqttConnectOptions connOpts = new MqttConnectOptions();
    connOpts.setCleanSession(true);

    client.connect(connOpts);
    MqttMessage message = new MqttMessage("Hello IoT".getBytes());
    client.publish("sensor/temperature", message);
    client.disconnect();
  }
}
MQTT IS THE DE FACTO STANDARD FOR IOT MESSAGING DUE TO ITS LIGHTWEIGHT NATURE AND EFFICIENCY

COAP (CONSTRAINED APPLICATION PROTOCOL)

CoAP is designed for constrained devices and networks, using UDP instead of TCP for lower overhead.

COAP VS HTTP
  • Transport: Uses UDP (fast, unreliable) vs. TCP (slower, reliable)
  • Request Model: GET, POST, PUT, DELETE similar to HTTP
  • Message Size: Optimized for small packets (4-byte header)
  • Multicast: Native support for broadcasting messages
  • Discoverability: Resource discovery via multicast

WEBSOCKETS

WebSockets provide full-duplex communication over a single TCP connection, ideal for real-time IoT applications.

WEBSOCKET ADVANTAGES
  • Real-time: Instant bidirectional communication
  • Efficient: Single connection, minimal overhead
  • Browser Support: Native in modern web browsers
  • Low Latency: Immediate message delivery
