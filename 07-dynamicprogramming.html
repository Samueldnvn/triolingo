DP Foundations

Two Key Properties

  • Overlapping Subproblems: Same subproblems recur
  • Optimal Substructure: Optimal solution contains optimal sub-solutions

Memoization vs Tabulation

Memoization: Top-down, recursive with caching. Easier to implement. Recursion depth limit.

Tabulation: Bottom-up, iterative table building. O(1) space optimizations possible.

1D Dynamic Programming

Climbing Stairs (Fibonacci)

dp[n] = dp[n-1] + dp[n-2]. dp[0] = 1, dp[1] = 2 (or dp[0]=0, dp[1]=1 depending on definition)

Coin Change (Minimum Coins)

dp[amount] = min(dp[amount], dp[amount-coin] + 1) for each coin

House Robber

rob[i] = max(rob[i-1], rob[i-2] + money[i])

2D Dynamic Programming

0/1 Knapsack

dp[i][w] = max(dp[i-1][w], dp[i-1][w-wt[i]] + val[i]). Space optimize to O(W) with reverse iteration.

Longest Common Subsequence

If match: 1 + dp[i-1][j-1]. Else: max(dp[i-1][j], dp[i][j-1])

Edit Distance

Match: dp[i-1][j-1]. Else: 1 + min(insert, delete, replace)

Advanced Topics

Bitmask DP

State includes bitmask representing subset (bits = elements included). Used for TSP, assignment problems.

LIS O(n log n) Optimization

Use binary search on tails array. tails[i] = min ending value of LIS of length i+1.
