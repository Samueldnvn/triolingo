Greedy Fundamentals

When Greedy Works

  • Problem has greedy choice property: locally optimal leads to global optimum
  • Optimal substructure: contains optimal sub-solutions

Common Greedy Problems

Activity Selection: Sort by finish time, pick earliest non-overlapping

Job Sequencing: Sort by profit, schedule at latest possible slot before deadline

Fractional Knapsack: Sort by value/weight ratio, take greedily

Graph Greedy Algorithms

Dijkstra

Shortest path, single-source, non-negative weights. Use priority queue. Always pick closest unvisited node.

Prim (MST)

Start from any vertex. Repeatedly add minimum edge connecting tree to new vertex. Use priority queue.

Kruskal (MST)

Sort all edges by weight. Process in order, add edge if connects different components (use Union-Find).
