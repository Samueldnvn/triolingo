🔨 HANDS-ON PROJECTS

Build your way from simple circuits to complete computers. Each project reinforces what you've learned with practical application.

BEGINNER PROJECT 1

3-Bit Comparator

EASY ⏱️ 2-3 hours

Build a circuit that compares two 3-bit binary numbers and outputs three signals: Equal (A=B), Greater Than (A>B), and Less Than (A<B).

🎯 Learning Objectives

  • Apply XOR gates for equality checking
  • Use AND gates with priority for comparison
  • Understand bit-by-bit comparison logic
  • Practice compact circuit design
  • Learn signal combination techniques
Requirements
  • Input: Two 3-bit numbers (6 input total)
  • Output: 3 signals (EQ, GT, LT)
  • Behavior: One output active at a time based on comparison result
Hints
  • Use XNOR gates to check if each bit pair is equal
  • AND all XNOR outputs to determine if A = B
  • For A > B, check high-order bits first
  • When bits are equal, check the next bit
  • A < B is the inverse of A > B (assuming equality check)
Challenges
  • Can you fit it in 5×5×5 blocks?
  • Can you achieve 4 tick delay or less?
  • Try building it vertically to save footprint
BEGINNER PROJECT 2

4-Bit Binary Calculator

EASY ⏱️ 3-4 hours

Create a calculator that adds two 4-bit binary numbers and displays the result.

🎯 Learning Objectives

  • Build a ripple-carry adder from full adders
  • Cascade multiple adder stages
  • Handle carry propagation
  • Connect inputs to circuit outputs
  • Display binary results visually
Requirements
  • Input: Two 4-bit numbers via buttons/switches
  • Circuit: 4-bit ripple-carry adder
  • Output: 5-bit result displayed (4 bits + 1 carry)
Stages
1
Build Full Adder

Create a single full adder using two half adders or direct gates

2
Cascade 4 Full Adders

Connect carry-out of each to carry-in of next

3
Add Inputs and Outputs

Connect input switches and output lamps

4
Test All Combinations

Verify results from 0 + 0 to 15 + 15

INTERMEDIATE PROJECT 3

8-Bit ALU

MEDIUM ⏱️ 6-8 hours

Build a complete 8-bit Arithmetic Logic Unit with multiple operations and flag outputs.

🎯 Learning Objectives

  • Design parallel operation circuits
  • Implement arithmetic and logic operations
  • Create MUX-based output selection
  • Generate control signals
  • Build flag detection circuits
Requirements
  • Inputs: Two 8-bit data values, 4-bit opcode
  • Operations: ADD, SUB, AND, OR, XOR, NOT (minimum 6)
  • Outputs: 8-bit result, 4 flags (Zero, Carry, Overflow, Sign)
  • Performance: Acceptable delay (don't worry about optimization)
ADD A + B
SUB A - B
AND A AND B
OR A OR B
XOR A XOR B
NOT NOT A
Building Strategy
  • Build 8 adders in vertical column (carry cascades down)
  • Build logic operations in horizontal rows (parallel)
  • Use 16-1 MUX to select one of 16 operation results
  • Add flag circuits at final output stage
  • Compact layout: ~30 height, ~80 width
INTERMEDIATE PROJECT 4

4-Bit Memory System

MEDIUM ⏱️ 5-7 hours

Create a functional RAM and ROM memory system with address decoding.

🎯 Learning Objectives

  • Understand address decoders
  • Build flip-flop registers for RAM cells
  • Implement ROM using torch placement
  • Create read/write control logic
  • Design shared data bus architecture
Requirements
  • RAM: 16 locations × 4 bits (64 bits total)
  • ROM: 16 locations × 4 bits (programmable)
  • Address: 4-bit address bus
  • Control: Read/Write, Chip Enable signals
Implementation
  • RAM: Use D flip-flops per bit cell, decoder selects word
  • ROM: Pre-set redstone torches, decoder activates row
  • Read: Selected location drives data bus
  • Write: Data bus drives into selected RAM location
Testing
  • Program a simple pattern into ROM
  • Read and verify each location
  • Write to RAM, read back to confirm
  • Test random access - read/write any location quickly
ADVANCED PROJECT 5

Control Unit with 8 Instructions

HARD ⏱️ 8-10 hours

Build a complete control unit that executes instructions following the fetch-decode-execute cycle.

🎯 Learning Objectives

  • Implement fetch-decode-execute cycle
  • Design instruction decoder
  • Create control signal generation
  • Build program counter with increment/jump
  • Synchronize all components with clock
Required Instructions
  • NOP: No operation
  • MOV: Move register to register
  • ADD: Add registers
  • SUB: Subtract registers
  • AND: Bitwise AND
  • JMP: Unconditional jump
  • JZ: Jump if zero flag set
  • OUT: Output to display
Cycle Implementation
  • Fetch: PC → Memory → Instruction Register
  • Decode: IR → Control Unit
  • Execute: Control signals trigger appropriate components
  • Repeat: PC increments for next instruction
⭐ FINAL PROJECT ⭐

Complete 8-Bit Computer

EPIC ⏱️ 30-60 hours

The masterpiece project. Build a fully functional 8-bit computer that runs programs. This is the culmination of everything you've learned.

🎯 Learning Objectives

  • Integrate all computer components
  • Design complete system architecture
  • Debug complex multi-component systems
  • Optimize performance and layout
  • Create and run real programs
Complete Requirements
CPU 8-bit ALU, Control Unit, 4 GPRs
Memory 256 bytes ROM (program), 256 bytes RAM (data)
I/O 8-bit button input, 2-digit hex display
Instructions Minimum 8 instructions, preferably 12+
Clock Configurable speed, proper distribution
Buses Data bus, Address bus, Control bus
Required Test Programs
  • Test 1: Add two input numbers, display result
  • Test 2: Count from 0 to 255 in a loop
  • Test 3: Fibonacci sequence generator
  • Test 4: Sort a list of numbers (advanced)
  • Test 5: Guess the number game (if-else logic)
Extra Credit Opportunities
  • Expand to 16-bit architecture
  • Add graphics display (pixel-based)
  • Implement multiplication/division hardware
  • Build a simple assembler or compiler
  • Add interrupt handling
  • Create a game or application
  • Optimize for minimal tick delay
  • Document and share your design
Building Phases
1
CPU Core

Build ALU, registers, decoder. Test independently

2
Memory System

Build ROM and RAM. Connect to CPU

3
I/O Integration

Add input panel and display

4
System Testing

Run test programs, debug, optimize

5
Polish & Document

Finalize layout, document everything, celebrate!

💡 Success Tips
  • Test each component separately before integrating
  • Plan cable management carefully - it will be a mess otherwise
  • Document your circuit with screenshots notes
  • Save schematic backups after major milestones
  • Use slow clock speed initially for debugging
  • Don't be afraid to redesign things - computers are complex!
  • Most importantly: Have patience and have fun!

Good Luck!

These projects are challenging but incredibly rewarding. Take your time, ask questions, and most importantly - enjoy building!
