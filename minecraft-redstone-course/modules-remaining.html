MODULE 2

ADVANCED CIRCUITRY

Move beyond basic gates into combinational logic. Build adders, multiplexers, and clocks - the foundation of digital systems.

⏱️ Estimated Time: 5-7 hours

🎯 Learning Objectives

  • Understand combinational vs sequential circuits
  • Build half adders, full adders, and ripple-carry adders
  • Create multiplexers and demultiplexers for data routing
  • Design encoders and decoders for signal interpretation
  • Generate reliable clock signals and timing circuits

2.1 Combinational vs Sequential Circuits

Combinational Circuits: Output depends only on current inputs. No memory of previous states. Examples: adders, MUX, decoders.

Sequential Circuits: Output depends on inputs AND previous state. Contains memory elements. Examples: flip-flops, counters, registers (covered in Module 5).

This module focuses on combinational circuits. They're immediate, predictable, and form the backbone of arithmetic and data flow operations.

2.2 Adders - The Heart of Computing

Adders perform binary addition. Let's build them step by step, starting simple and then scaling up.

Half Adder

Adds two 1-bit numbers (A and B). Produces a Sum bit and a Carry-out bit.

ABSumCarry
0000
0110
1010
1101
Half Adder Construction

Logic: Sum = A XOR B, Carry = A AND B

  • Sum output uses one XOR gate (check if bits differ)
  • Carry output uses one AND gate (both bits = 1)
  • Delay: 2 ticks (XOR delay)

Full Adder

Adds three 1-bit numbers: A, B, and Carry-In (from previous adder). Produces Sum and Carry-out.

Full Adder Construction Methods

Method 1: Two Half Adders Cascaded

  • First half adder: Add A + B to get partial sum and carry
  • Second half adder: Add partial sum + Carry-In
  • Carry-out = OR of both half adder carries
  • Delay: 4 ticks

Method 2: Direct Gate Network

  • Sum = A XOR B XOR Carry-In
  • Carry-Out = (A AND B) OR (A AND Carry-In) OR (B AND Carry-In)
  • More gates, but same delay and sometimes more compact

Ripple-Carry Adder

Cascades full adders to add multi-bit numbers. The carry "ripples" from least significant bit to most significant bit.

4-Bit Ripple-Carry Adder
  • Cascade 4 full adders
  • Each adder's Carry-Out → next adder's Carry-In
  • Total delay: 4 ticks per adder × 4 adders = 16 ticks!
  • This cascading delay is why we eventually use carry-lookahead adders

2.3 Multiplexers (MUX)

A multiplexer selects one of many inputs and routes it to a single output based on selector lines. Think of it as a digital switch.

2-to-1 Multiplexer

Two data inputs (I0, I1), one selector (S), one output (Y).

Selector (S)Output (Y)
0I0
1I1
2-to-1 MUX Implementation

Logic: Y = (NOT S AND I0) OR (S AND I1)

  • Use inverter on selector for NOT S
  • AND gates combine each input with appropriate selector
  • OR gate selects the combined result
  • Delay: 3 ticks

4-to-1 Multiplexer

Four data inputs, two selector lines (S1, S0), one output.

S1S0Output
00I0
01I1
10I2
11I3
4-to-1 MUX Construction

Method: Cascaded 2-to-1 MUX

  • First level: Two 2-to-1 MUX select between I0/I1 and I2/I3 using S0
  • Second level: One 2-to-1 MUX selects between first-level outputs using S1
  • Clean hierarchical design, easy to understand
  • Delay: 6 ticks (3 per level)
💡 MUX Applications
  • ALU input selection (which operation to perform)
  • Data bus routing (connecting components)
  • Implementing Boolean functions by tying inputs to constants
  • Building programmable logic arrays

2.4 Demultiplexers (DEMUX)

The opposite of MUX: takes one input and routes it to one of many outputs based on selector lines. Think of it as a signal distributor.

1-to-4 Demultiplexer

One data input, two selectors, four outputs.

Logic: Use decoder to enable specific output

  • 2-to-4 decoder converts selector lines to four enable signals
  • Each output = Input AND its enable signal
  • Only selected output receives the input signal
  • Useful for addressing memory or selecting devices

2.5 Decoders and Encoders

Decoder: Converts N input lines to 2^N output lines. Exactly ONE output is active based on input encoding.

Encoder: Converts 2^N input lines to N output lines. Determines which input is active and outputs its code.

2-to-4 Decoder

Decoder Implementation
  • Each output is an AND gate checking for specific input combination
  • Y0 = NOT I1 AND NOT I0
  • Y1 = NOT I1 AND I0
  • Y2 = I1 AND NOT I0
  • Y3 = I1 AND I0
  • Used in memory addressing, display drivers, instruction decoding

2.6 Clock Generators and Timing

Clocks provide periodic, regular pulses that synchronize circuit operation. Essential for sequential computers.

Clock Generator Designs

1. Classic Clock (Pulse Generator)

  • Empty block → repeater → empty block → repeater (cycle)
  • Repeater delay determines clock period
  • 1-click repeater = 1 tick pulse every 4 ticks
  • Standard, reliable, but slower

2. Hopper Clock (Compact)

  • Two hoppers facing each other, passing items
  • Comparator detects hopper contents
  • Period depends on item transfer speed
  • Very compact and customizable

3. Observer Clock (Fast)

  • Two observers facing each other
  • Observed blocks update (e.g., plants, flowing water)
  • Can achieve 1 tick or faster periods
  • Fastest but least predictable

4. Comparator Loop Clock (Variable)

  • Comparator in loop with container and comparator
  • Adjustable period via item count in container
  • Most flexible for timing adjustments
💡 Clock Design Tips
  • Always isolate your clock from circuit feedback
  • Add emergency stop (redstone torch to break the loop)
  • Use multiple clock speeds in one computer (fast clock for ALU, slow for display)
  • Consider edge-triggered vs level-triggered circuits
  • Remember: Minecraft ticks = 0.1 seconds real time

BINARY SYSTEMS

Master the number system that computers use. Learn binary representation, arithmetic, and how to work with negative numbers.

⏱️ Estimated Time: 3-5 hours

🎯 Learning Objectives

  • Understand why computers use binary
  • Convert decimal ↔ binary ↔ hexadecimal
  • Perform binary addition, subtraction, multiplication, division
  • Represent negative numbers using two's complement
  • Work with signed and unsigned binary numbers
  • Understand binary fractions and floating-point concepts

3.1 Why Binary?

Computers use binary because hardware devices are inherently two-state: ON/OFF, HIGH/LOW, 1/0. It's reliable, simple, and scales perfectly to complex operations.

In Minecraft redstone, this is natural: signal is either present (1-15) or absent (0). We treat any signal as "1" and no signal as "0".

💡 Binary vs Decimal
  • Decimal (base-10): 0-9 digits, powers of 10
  • Binary (base-2): 0-1 digits, powers of 2
  • Same value, different representation
  • Example: 13 decimal = 8 + 4 + 1 = 1101 binary

3.2 Binary Number System

In binary, each position represents a power of 2. From right to left: 1, 2, 4, 8, 16, 32, 64, 128, etc.

Reading Binary Numbers

Example: 1101 (4-bit)
Position:  3  2  1  0
Value:   2^3 2^2 2^1 2^0
Weight:   8  4  2  1
Bits:     1  1  0  1

Sum = (1×8) + (1×4) + (0×2) + (1×1) = 13 (decimal)

Binary place values for N bits

  • 4 bits: 0-15 (values)
  • 8 bits: 0-255
  • 16 bits: 0-65,535
  • Our computer will use 8 bits = 1 byte

3.3 Decimal to Binary Conversion

Method 1: Divide by 2, collect remainders

Convert 25 to binary:

25 ÷ 2 = 12 remainder 1  (least significant bit)
12 ÷ 2 =  6 remainder 0
 6 ÷ 2 =  3 remainder 0
 3 ÷ 2 =  1 remainder 1
 1 ÷ 2 =  0 remainder 1  (most significant bit)

Result: 11001 (read remainders bottom to top)

Method 2: Subtract powers of 2

Convert 25 to binary:

Largest power of 2 ≤ 25 is 16 (2^4)
  16  8  4  2  1   (powers)
   1  ?  ?  ?  ?   (25-16=9, continue)

  9-8=1, remaining 1
  1-1=0, remaining 0

Fill in missing powers with 0:

  16  8  4  2  1
   1  1  0  0  1

Result: 11001

3.4 Binary Arithmetic

Binary Addition

  1011  (11 decimal)
+ 0101  (5  decimal)
-------
 10000  (16 decimal)

Carry: 111
  • Add bit by bit, right to left
  • 0+0=0, 1+0=1, 1+1=0 carry 1, 1+1+1=1 carry 1
  • This is exactly what adder circuits do!

Binary Subtraction

  1010  (10 decimal)
- 0110  (6  decimal)
-------
  0100  (4  decimal)

Borrow: 01
  • Subtract bit by bit, right to left
  • 0-0=0, 1-0=1, 1-1=0, 0-1 needs borrow
  • Hardware uses two's complement (makes subtraction = addition)

Binary Multiplication

   101  (5 decimal)
×  011  (3 decimal)
-------
   101  (5×1)
  101   (5×2, shifted)
+000    (5×0, shifted)
-------
  1111  (15 decimal)
  • Binary multiplication is just repeated addition and shifting
  • "AND" each bit of multiplier, shift, and sum results
  • This is how computers multiply!

Binary Division

Divide 1101 (13) by 10 (2):

  • Long division in binary
  • Successive subtraction and shifting
  • Complex circuit but follows same principles

3.5 Signed Numbers - Two's Complement

How do computers represent negative numbers? The answer: two's complement.

Two's Complement Rules

To get two's complement of a number:

  1. Write number in binary
  2. Invert all bits (NOT operation)
  3. Add 1
Example: Represent -5 in 4-bit two's complement
Step 1: Write +5 in binary
    0101

Step 2: Invert all bits
    1010

Step 3: Add 1
    1010
+   1
-------
    1011  ← This is -5 in two's complement!

Range of Signed Numbers

  • 4-bit signed: -8 to +7
  • 8-bit signed: -128 to +127
  • 16-bit signed: -32,768 to +32,767
  • Most significant bit is sign bit (1 = negative, 0 = positive)

Subtraction via Two's Complement

Calculate: 7 - 5 = 2

Method: 7 + (-5)

Write 7 in binary:        0111
Write -5 in two's comp:   1011
                                                        
Add:                     0111
                        +1011
                        -----
                       10010  (carry out, discard)

Result: 0010 = 2 decimal ✓
💡 Why Two's Complement?
  • Subtraction becomes addition (same hardware!)
  • Unique representation of zero (not +0 and -0)
  • Overflow detection is simple
  • Most significant bit automatically indicates sign
  • Standard in all computers

MODULE 4

ALU DESIGN

Build an Arithmetic Logic Unit - the calculator heart of your computer. Perform math and bitwise operations.

⏱️ Estimated Time: 6-8 hours

🎯 Learning Objectives

  • Understand ALU architecture and control signals
  • Build arithmetic operations: add, subtract, increment, decrement
  • Implement bitwise operations: AND, OR, XOR, NOT, shift
  • Create operation selector circuits
  • Handle flags: zero, carry, overflow, sign
  • Design compact, efficient ALU layouts

4.1 What is an ALU?

The Arithmetic Logic Unit performs arithmetic (add, subtract) and logic (AND, OR, XOR, etc.) operations on data. It's controlled by operation codes that specify which operation to perform.

Our ALU will be 8-bit with 16 different operations.

4.2 ALU Architecture

Inputs: Two 8-bit data values (A, B)

Control: 4-bit operation code (allows 16 operations)

Outputs: 8-bit result + 4 flag bits (zero, carry, overflow, sign)

Internal: Parallel operating units selected by MUX

4.3 Arithmetic Operations

8-Bit Adder

  • 8 cascaded full adders
  • Carry ripple from LSB to MSB
  • Faster variants: carry-lookahead, carry-skip (advanced)

Subtraction

  • Subtract = Add + Twos Complement of B
  • Input B: XOR with carry-in (bitwise NOT when Cin=1)
  • Set carry-in to 1 to add 1 complete two's complement
  • Reuse adder hardware for subtraction - efficient!

Increment/Decrement

  • Increment: Add A + 1
  • Decrement: Add A + (-1)
  • Special optimization: Only need propagate carry for one addition

4.4 Bitwise Operations

Bitwise operations operate on each bit independently. Very fast!

Operations

  • AND: Each bit = A AND B (bitwise)
  • OR: Each bit = A OR B (bitwise)
  • XOR: Each bit = A XOR B (bitwise)
  • NOT: Each bit = NOT A (bitwise, only needs one input)
  • Shifts: Left shift = multiply by 2, Right shift = divide by 2
Bitwise Operation Example
A = 10110011
B = 11001010

A AND B = 10000010  (bit-by-bit AND)
A OR  B = 11111011  (bit-by-bit OR)
A XOR B = 01111001  (bit-by-bit XOR)
NOT A   = 01001100  (bitwise NOT)

A << 1  = 01100110  (left shift)
A >> 1  = 01011001  (right shift)

4.6 ALU Flags

Flags provide conditional information about the result:

  • Zero Flag (Z): Result = 0? (OR all output bits together)
  • Carry Flag (C): Did addition overflow? (Last adder's carry-out)
  • Overflow Flag (V): Signed overflow? (XOR of last two carry bits)
  • Sign Flag (S): Result negative? (Most significant bit)

These flags are crucial for conditional jumps and comparisons in programs!

4.7 ALU Circuit Layout

Optimized Layout Strategy
  • Layer 1: Bitwise operations (parallel, no carry propagation)
  • Layer 2: Adder/Subtractor (vertical cascade for carry)
  • Layer 3: Shift registers (rotate bits)
  • Layer 4: MUX tree to select operation output
  • Layer 5: Flag generation circuits

Total height: ~30 blocks. Width: ~60 blocks. Compact!

5.1 Introduction to Memory

Sequential circuits have memory - they can store information between clock cycles. Memory is built from bistable elements that have two stable states they can maintain indefinitely.

5.2 RS Latch

Simplest memory element. Set input (S) stores 1, Reset input (R) stores 0.

RS Latch Implementation

  • Each gate's output feeds the other's input
  • Feedback creates memory
  • Level-sensitive, needs gate inputs to hold state

D Flip-Flop Uses

  • Pipeline stages - pass data between stages
  • Data sampling - capture input on clock

Toggle flip-flop. On clock edge when T=1, output toggles. T=0, no change.

Perfect for binary counters! Each T-FF toggles when lower bit goes 1→0.

5.6 Registers

A register stores multi-bit values. Essentially an array of D flip-flops.

8-Bit Register

  • 8 D flip-flops in parallel
  • Same clock triggers all
  • Store 8-bit values
  • Can have write-enable control
Register with Write-Enable
  • Only update on clock when WE = 1
  • When WE = 0, maintain value
  • Use clock gating or D-FF enable
  • Essential for selective memory writes

5.7 RAM Design

RAM (Random Access Memory) provides random read/write to any location. Key concept: address decoding.

Memory Organization

  • 8-bit address → 256 memory locations
  • 8-bit data width (our computer)
  • Total memory: 256 bytes = 2,048 bits of storage

RAM Operations

  • Read: Address selects location, data appears at output
  • Write: Address selects location, data written to location
  • Control: Read/Write signal, Chip Enable
RAM Implementation Strategy
  • Decoder converts 8-bit address to 256 word-lines
  • Each word-line activates one register
  • Read: Selected register's data drives output
  • Write: Input data driven into selected register
  • In Minecraft, use redstone blocks for compact storage

5.8 ROM Design

ROM (Read Only Memory) stores fixed data or programs. Cannot be modified during operation (except at design time).

Flash ROM in Minecraft

  • Preset redstone torches for data cells (1)
  • No torch = 0
  • Address decoder selects row
  • Data from selected row appears at output
  • To program: break/place torches during construction

6.1 Computer Instruction Set

An instruction set defines all operations a computer can perform. We'll design a CISC-style instruction set with 16 instructions.

Instruction Format

  • Opcode: 4 bits (defines operation)
  • Register A: 2 bits (source/destination)
  • Register B: 2 bits (source)

6.2 Instruction Decoder

Instruction decoder takes opcode and generates control signals for rest of computer.

Decoder Architecture
  • 4-bit opcode input
  • 4-to-16 decoder generates 16 signals (one per instruction)
  • Each instruction's signals are pre-wired to appropriate components
  • Example: ADD instruction triggers ALU to add, registers to read

6.3 Fetch-Decode-Execute Cycle

Every instruction goes through three phases:

1. Fetch

  • Program Counter (PC) points to instruction
  • Memory at PC address outputs instruction
  • Instruction loaded into Instruction Register (IR)
  • PC incremented by 1

2. Decode

  • IR output goes to instruction decoder
  • Decoder identifies operation
  • Control signals generated

3. Execute

  • Control signals trigger appropriate components
  • ALU performs arithmetic/logic
  • Memory accessed if load/store
  • Results stored or output
  • Flags updated

Then cycle repeats for next instruction!

6.4 Program Counter

Program Counter holds address of next instruction. Special register with:

  • Increment: Add 1 each fetch
  • Load: Set to jump target address
  • Conditional Load: Load only if flag condition met
PC Implementation
  • Output drives memory address decoder
  • Input from jump target addresses
  • MUX selects between increment PC vs jump address
  • Controlled by fetch/decode/jump signals

6.5 Conditional Jumps

Conditional jumps allow programs to make decisions (if statements, loops).

Implementation

  • Check relevant flag (Zero, Carry, Sign, Overflow)
  • If flag matches condition, enable PC load
  • Jump target address provided in instruction or register
JZ (Jump if Zero) Logic
  • Zero Flag AND = 1 when instruction is JZ flag is set
  • This signal enables PC load
  • If flag = 0, PC loads from increment (normal execution)
  • Essential for loops and if statements!

Decimal Display

8-Bit Hex Display

7.3 Data Persistence

Store data even when computer off or chunk unloads.

Item-Based Storage

  • Hoppers contain specific items = data value
  • Item count = data value (1-64)
  • Item type can distinguish multiple bytes
  • Persists across saves but slow to access

Block State Storage

  • Piston extension state = 1 bit
  • Multiple pistons = multi-bit storage
  • Instant access but complex wiring, persists only while loaded

  • Buses: Data bus (8-bit), Address bus (8-bit), Control bus (multiple signals)

8.2 Bus Design

Buses connect components. Need careful design.

Data Bus (8-bit)

  • Carries data between CPU, memory, I/O
  • Tri-state buffers allow multiple sources
  • Direction control for read vs write

Address Bus (8-bit)

  • Carries memory/RAM address
  • One-way (CPU → memory)
  • Driven by PC or load/store address

Control Bus (multi-signal)

  • Clock (synchronizes all components)
  • Read/Write (memory operation type)
  • Operation codes (ALU control)
  • Flags (conditional jumps)

8.3 Clock Distribution

Single clock signal distributed to all sequential components.

  • CPU: Triggers fetch-decode-execute
  • Memory: Triggers read/write latches
  • Registers: Capture data on clock edge
  • I/O: Sample input on clock edge

Challenge: All components must see same clock simultaneously!

9.1 Pipelining

Overlapping instruction execution for speed. While instruction 1 executes, instruction 2 is decoded, instruction 3 is fetched.

3-Stage Pipeline

  • Stage 1: Fetch
  • Stage 2: Decode
  • Stage 3: Execute

In Minecraft, pipelining is complex due to space constraints and signal delay. More theoretical for understanding real computers.

9.2 Interrupts

Interrupts allow external events to pause execution and run interrupt handlers.

Implementation

  • Interrupt line signals CPU
  • CPU completes current instruction
  • Saves PC state
  • Jumps to interrupt handler address
  • Handler returns to original execution

Useful for: timers, user input, external events

9.3 CPU Cache

Small, fast memory that holds frequently accessed data. Reduces slow main memory accesses.

Cache Levels

  • L1 Cache: Smallest, fastest, inside CPU
  • L2 Cache: Larger, slightly slower
  • Main Memory: RAM

In Minecraft, cache is rarely practical due to space constraints. Understanding the concept is valuable.
