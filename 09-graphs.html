Graph Representations

Adjacency List

Most common in competitive programming. O(V + E) space. Efficient for sparse graphs.

graph = {node: [neighbors]} or graph = [[] for _ in range(n)]

Adjacency Matrix

O(V^2) space. O(1) edge check. Good for dense graphs or when need fast edge queries.

BFS and DFS

BFS (Breadth-First Search)

Queue-based, level-order. Shortest path in unweighted graphs. O(V + E).

DFS (Depth-First Search)

Stack or recursive. Explores deeply. Used for cycle detection, topological sort, connected components.

Topological Sort

Kahn: Remove zero-indegree vertices. DFS: Add after fully exploring (post-order). Only for DAGs.

Shortest Path Algorithms

Dijkstra

Single-source, non-negative weights. O((V + E) log V) with binary heap.

Bellman-Ford

Handles negative weights, detects negative cycles. O(V * E). Relax all edges V-1 times.

Floyd-Warshall

All-pairs shortest paths. O(V^3). Good for dense graphs or all pairs needed.

MST and Advanced Topics

Minimum Spanning Tree

Prim: Grow from vertex, priority queue of edges. O((V + E) log V)

Kruskal: Sort edges, Union-Find. O(E log E)

Union-Find Disjoint Set Union

Path compression + Union by rank makes operations nearly O(1) amortized.
