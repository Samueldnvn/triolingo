Greedy vs DP

Greedy: make locally optimal choice, hope for global optimum. Fast but doesn't always work. DP: explores all possibilities, guarantee optimal solution. Slower but always correct. Key question: does greedy choice property hold?

Greedy Fundamentals

Activity Selection

def activity_selection(start, finish):
    """Select max non-overlapping activities"""
    activities = sorted(zip(start, finish), key=lambda x: x[1])
    count = 0
    end_time = float('-inf')
    
    for s, f in activities:
        if s >= end_time:
            count += 1
            end_time = f
    
    return count

# Sort by finish time, select if starts after previous ends
# This is optimal: proof by exchange argument

Jump Game

def can_jump(nums):
    """Check if can reach last index"""
    max_reach = 0
    for i, num in enumerate(nums):
        if i > max_reach:
            return False
        max_reach = max(max_reach, i + num)
    return True

# Greedy: track furthest we can reach

Minimum Number of Arrows to Burst Balloons

def find_min_arrow_shots(points):
    """Arrows needed to burst all balloons"""
    if not points: return 0
    
    # Sort by end coordinate
    points.sort(key=lambda x: x[1])
    
    arrows = 1
    end = points[0][1]
    
    for start, finish in points[1:]:
        if start > end:
            arrows += 1
            end = finish
    
    return arrows

Common Greedy Problems

Assign Cookies

def find_content_children(g, s):
    """Max number of content children"""
    g.sort()  # Children greed factor
    s.sort()  # Cookie sizes
    
    child_i = cookie_j = 0
    while child_i < len(g) and cookie_j < len(s):
        if s[cookie_j] >= g[child_i]:
            child_i += 1
        cookie_j += 1
    
    return child_i

Partition Labels

def partition_labels(s):
    """Partition string into minimum parts"""
    last = {c: i for i, c in enumerate(s)}
    result = []
    start = end = 0
    
    for i, c in enumerate(s):
        end = max(end, last[c])
        if i == end:
            result.append(end - start + 1)
            start = i + 1
    
    return result
GREEDY PROPERTY: Optimal solution can be constructed by making optimal local choices. Proof by exchange argument usually needed.
