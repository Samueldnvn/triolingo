IoT Application Development Overview

Building Complete End-to-End IoT Systems

Building a complete IoT application requires understanding the entire system: devices, connectivity, cloud, data processing, and user interfaces. This module ties together concepts from previous modules to guide you through designing, implementing, and deploying real-world IoT applications.

IoT System Architecture

Designing Scalable and Reliable IoT Systems

Good architecture separates concerns, enables scalability, and supports evolution. IoT architectures range from simple device-to-cloud to complex hierarchies with edge computing and multiple regional deployments.

Architecture Patterns

COMMON IOT ARCHITECTURES
  • Device-to-Cloud: Simplest, all devices connect directly to cloud. Good for low device counts, low-latency requirements.
  • Gateway-Aggregated: Local gateway aggregates multiple devices, single cloud connection. Good for home/building IoT.
  • Edge-Cloud Hybrid: Some processing at edge, rest in cloud. Good for bandwidth optimization, offline capability.
  • Multi-Cloud: Multiple cloud providers for redundancy or regulatory reasons.

Connecting Components Into a Coherent System

Integration involves ensuring all components work together: devices send data correctly, cloud processes and stores it, APIs expose it for consumption, and applications present it to users. APIs and message queues are the glue that holds IoT systems together.

import org.springframework.web.bind.annotation.*;
import org.springframework.beans.factory.annotation.Autowired;

@RestController
@RequestMapping("/api/iot")
public class IoTDataController {
    
    @Autowired
    private IoTDataService dataService;
    
    @GetMapping("/devices/{deviceId}/readings")
    public List<SensorReading> getDeviceReadings(
            @PathVariable String deviceId,
            @RequestParam String startTime,
            @RequestParam(required = false) String endTime) {
        
        ZonedDateTime start = ZonedDateTime.parse(startTime);
        Optional<ZonedDateTime> end = Optional.ofNullable(endTime)
            .map(ZonedDateTime::parse);
            
        return dataService.getReadings(deviceId, start, end);
    }
    
    @GetMapping("/devices/{deviceId}/latest")
    public SensorReading getLatestReading(
            @PathVariable String deviceId) {
        return dataService.getLatestReading(deviceId);
    }
    
    @GetMapping("/devices/{deviceId}/history/{metric}")
    public List<DataPoint> getMetricHistory(
            @PathVariable String deviceId,
            @PathVariable String metric,
            @RequestParam String aggregation,
            @RequestParam long periodMs) {
        
        return dataService.getAggregatedData(
            deviceId, metric, aggregation, periodMs
        );
    }
}

Message Queue for Event-Driven Architecture

KAFKA PRODUCER FOR IOT EVENTS
import org.apache.kafka.clients.producer.*;
import java.util.Properties;

public class IoTEventProducer {
    private Producer<String, String> producer;
    
    public void init() {
        Properties props = new Properties();
        props.put("bootstrap.servers", "kafka:9092");
        props.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
        props.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");
        
        producer = new KafkaProducer<>(props);
    }
    
    public void sendDeviceEvent(String deviceId, String eventType, Object payload) {
        ObjectMapper mapper = new ObjectMapper();
        IoTEvent event = new IoTEvent(
            deviceId,
            eventType,
            System.currentTimeMillis(),
            payload
        );
        
        String key = deviceId;
        String value = mapper.writeValueAsString(event.getPayload());
        
        ProducerRecord<String, String> record = new ProducerRecord<>(
            "iot-events",
            key,
            value
        );
        
        producer.send(record, (metadata, exception) -> {
            if (exception != null) {
                System.err.println("Failed to send event: " + exception);
            } else {
                System.out.println("Event sent to partition " + metadata.partition());
            }
        });
    }
}

class IoTEvent {
    private String deviceId;
    private String eventType;
    private long timestamp;
    private Object payload;
}

Real-Time Updates with WebSockets

SPRING WEBSOCKET FOR live UPDATES
import org.springframework.web.socket.*;
import org.springframework.web.socket.handler.TextWebSocketHandler;

@Component
public class IoTWebSocketHandler extends TextWebSocketHandler {
    
    private static ConcurrentHashMap<String, WebSocketSession> sessions 
        = new ConcurrentHashMap<>();
    
    @Override
    public void afterConnectionEstablished(WebSocketSession session) throws Exception {
        String deviceId = (String) session.getAttributes().get("deviceId");
        sessions.put(deviceId, session);
        System.out.println("Client connected for device: " + deviceId);
    }
    
    @Override
    public void afterConnectionClosed(WebSocketSession session, 
                                     CloseStatus status) throws Exception {
        String deviceId = (String) session.getAttributes().get("deviceId");
        sessions.remove(deviceId);
    }
    
    public void broadcastToClient(String deviceId, String message) {
        WebSocketSession session = sessions.get(deviceId);
        if (session != null && session.isOpen()) {
            try {
                session.sendMessage(new TextMessage(message));
            } catch (IOException e) {
                System.err.println("Failed to send message: " + e.getMessage());
                sessions.remove(deviceId);
            }
        }
    }
}

// Triggered when new sensor data arrives

}

Device Code: Temperature Sensor with MQTT

ANDROID THINGS (JAVA) TEMPERATURE SENSING
import com.google.android.things.pio.Gpio;
import com.google.android.things.contrib.driver.dhtxx.Dhtxx;
import org.eclipse.paho.client.mqttv3.*;
import java.nio.charset.StandardCharsets;

public class TemperatureSensorService extends Service {
    private Dhtxx dhtxxSensor;
    private MqttClient mqttClient;
    private String deviceId = "temp-sensor-001";
    
    @Override
    public void onCreate() {
        // Initialize DHT22 sensor
        dhtxxSensor = new Dhtxx("BCM4", Dhtxx.DHT_TYPE_22);
        
        // Connect to MQTT broker
        connectMQTT();
        
        // Start periodic reading (10 Hz)
        Handler handler = new Handler();
        Runnable readTask = new Runnable() {
            @Override
            public void run() {
                readAndPublish();
                handler.postDelayed(this, 100);
            }
        };
        handler.post(readTask);
    }
    
    private void connectMQTT() {
        try {
            mqttClient = new MqttClient(
                "tcp://mqtt.example.com:1883",
                deviceId
            );
            MqttConnectOptions options = new MqttConnectOptions();
            options.setCleanSession(true);
            options.setConnectionTimeout(30);
            
            mqttClient.connect(options);
        } catch (MqttException e) {
            System.err.println("MQTT connection failed");
        }
    }
    
    private void readAndPublish() {
        try {
            float[] values = dhtxxSensor.readValues();
            float temperature = values[0];
            float humidity = values[1];
            
            if (!Float.isNaN(temperature)) {
                TemperatureReading reading = new TemperatureReading(
                    deviceId,
                    temperature,
                    humidity,
                    System.currentTimeMillis()
                );
                
                String json = objectMapper.writeValueAsString(reading);
                mqttClient.publish(
                    "temperature/reading",
                    new MqttMessage(json.getBytes(StandardCharsets.UTF_8))
                );
            }
        } catch (Exception e) {
            System.err.println("Sensor read failed");
        }
    }
}

Control Logic: Temperature Regulation

SPRING RULE ENGINE
@Service
public class TemperatureControlService {
    
    @Value("${temp.heating-threshold:18}")
    private int heatingThreshold;
    
    @Value("${temp.cooling-threshold:26}")
    private int coolingThreshold;
    
    @Value("${temp.low-alert:15}")
    private int lowAlertThreshold;
    
    @Value("${temp.high-alert:30}")
    private int highAlertThreshold;
    
    private Map<String, TempState> deviceStates = new ConcurrentHashMap<>();
    
    @KafkaListener(topics = "temperature-readings")
    public void processTemperature(TemperatureReading reading) {
        String deviceId = reading.getDeviceId();
        TempState state = deviceStates.computeIfAbsent(
            deviceId, k -> new TempState()
        );
        
        double temp = reading.getTemperature();
        state.addReading(temp, System.currentTimeMillis());
        
        // Check alerts
        if (temp < lowAlertThreshold) {
            alertService.sendAlert(deviceId, "LOW_TEMP", 
                "Temperature " + temp + "°C below " + lowAlertThreshold + "°C");
        }
        if (temp > highAlertThreshold) {
            alertService.sendAlert(deviceId, "HIGH_TEMP",
                "Temperature " + temp + "°C above " + highAlertThreshold + "°C");
        }
        
        // Control logic (based on sustained state for 5 minutes)
        if (state.hasLowTempFor(periodMs = 300000L)) {
            controlDevice(deviceId, "heater", true);
            controlDevice(deviceId, "ac", false);
        }
        else if (state.hasHighTempFor(periodMs = 300000L)) {
            controlDevice(deviceId, "ac", true);
            controlDevice(deviceId, "heater", false);
        }
        else {
            controlDevice(deviceId, "heater", false);
            controlDevice(deviceId, "ac", false);
        }
    }
    
    private void controlDevice(String deviceId, String device, boolean on) {
        ControlMessage msg = new ControlMessage(deviceId, device, on);
        mqttGateway.publish("device/control", msg);
    }
}

class TempState {
    private Queue<Double> readings = new LinkedList<>();
    private Queue<Long> timestamps = new LinkedList<>();
    
    public void addReading(double temp, long timestamp) {
        readings.add(temp);
        timestamps.add(timestamp);
    }
    
    public boolean hasLowTempFor(long periodMs) {
        if (timestamps.isEmpty()) return false;
        return timestamps.getLast() - timestamps.getFirst() >= periodMs &&
               readings.stream().allMatch(t -> t < 18);
    }
    
    public boolean hasHighTempFor(long periodMs) {
        if (timestamps.isEmpty()) return false;
        return timestamps.getLast() - timestamps.getFirst() >= periodMs &&
               readings.stream().allMatch(t -> t > 26);
    }
}

Anomaly Detection with ML

FLINK CEP + ML ANOMALY DETECTOR
public class PredictiveMaintenanceStream {
    
    private TensorFlowModel anomalyModel;
    
    public void setup() {
        // Load trained anomaly detection model
        anomalyModel = new TensorFlowModel("maintenance_model.pb");
    }
    
    public void detectAnomalies(DataStream<SensorReading> readings) {
        // Compute features: moving average, std dev, frequency spectrum
        DataStream<EquipmentFeatures> features = readings
            .keyBy(SensorReading::getEquipmentId)
            .window(SlidingEventTimeWindows.of(Time.minutes(5), Time.minutes(1)))
            .aggregate(new FeatureAggregator());
        
        // Run anomaly detection model
        DataStream<AnomalyScore> scores = features
            .map(new FeatureToAnomalyScore(anomalyModel));
        
        // Alert on high anomaly scores (> 0.8 threshold)
        DataStream<Alert> alerts = scores.filter(score -> score.getScore() > 0.8)
            .map(score -> new Alert(
                PRIORITY.HIGH,
                "EQUIPMENT_ANOMALY",
                "Anomaly detected for equipment " + score.getEquipmentId() + 
                " with score " + score.getScore(),
                score.getEquipmentId()
            ));
        
        scores.addSink(new MaintenanceOrderSink());
    }
}

class FeatureAggregator implements AggregateFunction {
    @Override
    public EquipmentFeatures add(SensorReading reading, FeatureState state) {
        state.add(reading);
        return state.toFeatures();
    }
    
    @Override
    public EquipmentFeatures getResult(FeatureState state) {
        return state.toFeatures();
    }
}

Alert and Maintenance Ordering

MAINTENANCE ORDER SYSTEM
@Service
public class MaintenanceService {
    
    @Autowired
    private MaintenanceOrderRepository orderRepo;
    
    @Autowired
    private SlackService slackService;
    
    @KafkaListener(topics = "maintenance-alerts")
    public void handleAlert(Alert alert) {
        System.out.println("Alert: " + alert.getMessage());
        
        // Check if maintenance order already exists
        Optional<MaintenanceOrder> existing = 
            orderRepo.findOpenByEquipmentId(alert.getEquipmentId());
        
        if (!existing.isPresent()) {
            // Create new maintenance order
            MaintenanceOrder order = new MaintenanceOrder(
                alert.getEquipmentId(),
                alert.getPriority(),
                alert.getMessage(),
                ScheduledStatus.PENDING
            );
            
            orderRepo.save(order);
            
            // Notify maintenance team
            slackService.sendAlert(
                "#maintenance",
                "New maintenance order: " + alert.getMessage() + 
                "\nEquipment: " + alert.getEquipmentId() +
                "\nOrder ID: " + order.getId()
            );
        }
    }
}

Deployment and Monitoring

Deploying and Operating IoT at Scale

Deploying IoT systems involves provisioning devices, configuring cloud infrastructure, setting up monitoring and alerting, and establishing maintenance procedures. For production systems, reliability and scalability are paramount.

DEPLOYMENT CHECKLIST
  • Device Provisioning: Unique certificates per device, initial configuration

Device Provisioning with Unique Certificates

DEVICE PROVISIONING WORKFLOW
// Manufacturing process for 1000 devices

1. Generate unique device certificate per device
   for i in {1..1000}
   do
     # Generate device certificate
     openssl genrsa -out device-$i.key 2048
     openssl req -new -key device-$i.key -out device-$i.csr \
         -subj "/OU=device-$i"
     openssl x509 -req -in device-$i.csr -CA ca.crt -CAkey ca.key \
         -CAcreateserial -out device-$i.crt -days 365
   done

2. Install certificate on device
   - Flash device-$i.crt and device-$i.key to device flash
   - Flash CA certificate for server validation

3. Register device in cloud
   POST /api/devices
   {
     "deviceId": "device-$i",
     "certificate": "...base64 of device-$i.crt...",
     "type": "temp-sensor",
     "location": "warehouse-floor-3"
   }

4. Device authenticates using certificate on first boot
   - MQTT client certificate authentication
   - Device receives configuration
   - Device registers as online

Practice Problems

Test Your Understanding of IoT Application Development

9-1 Easy Definition
What is the purpose of a device shadow in IoT?
A device shadow stores desired and reported state, enabling control and synchronization for devices that may be offline or with intermittent connectivity
9-2 Medium Concept
Why is a gateway typically used in IoT architectures?
Gateways aggregate multiple devices, perform protocol translation, provide edge processing, reduce cloud connections (cost and bandwidth), and enable offline operation
9-3 Easy Concept
True or False: WebSockets are appropriate for IoT device-to-cloud communication.
False - WebSockets are useful for client-to-server real-time updates (dashboards). For device-to-cloud, MQTT is preferred due to lower overhead and better support for intermittent connectivity.
9-4 Medium Application
You are building a smart lighting system with 1000 light bulbs that need real-time control. What architecture and technologies would you recommend?
Gateway-Aggregated Architecture: Zigbee mesh (lights are routers) → local gateway (Raspberry Pi) → MQTT → cloud. Use Zigbee for device communication (low power, mesh), MQTT for cloud (lightweight), WebSocket for real-time UI updates. Scale: multiple gateways for different floors/areas.
