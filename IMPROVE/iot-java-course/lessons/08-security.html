IoT Security Fundamentals

Protecting the Internet of Things from Attacks

IoT security is critical yet often overlooked due to devices' limited resources, physical accessibility, and scale. A single compromised device can serve as a foothold for attacking entire networks. This module covers fundamental security principles and practical implementation strategies for IoT systems.

WHY IOT SECURITY MATTERS
  • Scale: Attackers can weaponize millions of devices (Mirai botnet)
  • Physical Access: Devices often deployed in public locations
  • Limited Update: Hard patching = long vulnerabilities
  • Privacy: Cameras, microphones can invade homes
  • Safety: Compromised medical devices, autonomous vehicles can cause harm

CIA Triad for IoT

SECURITY PRINCIPLES
  • Confidentiality: Prevent unauthorized data access
  • Integrity: Ensure data not tampered with or corrupted
  • Availability: Ensure systems remain operational

THREAT MODEL

Assume devices will be attacked. Design for compromise: limit blast radius, detect intrusions, enable quick recovery. Security is a process, not a product.

Authentication & Authorization

Verifying Device Identity and Permissions

Authentication verifies identity (who are you?) while authorization determines permissions (what can you do?). IoT devices need both: mutual TLS for secure connections, device certificates for proof of identity, and access control for services.

AUTHENTICATION METHODS
  • Pre-Shared Keys (PSK): Simple, but key distribution is hard at scale
  • X.509 Certificates: Industry standard, supports certificate chains
  • OAuth 2.0: Token-based delegation, good for web/mobile
  • JWT (JSON Web Tokens): Self-contained, stateless, IoT-friendly
  • Biometrics: For user-device interaction (fingerprint, facial)

Device Authentication with Certificates

MUTUAL TLS WITH JAVA
import javax.net.ssl.*;
import java.security.KeyStore;
import java.io.FileInputStream;

public class MutualTLSClient {
    public SSLContext createMutualTLSContext() throws Exception {
        // Load device certificate and key
        KeyStore deviceKeystore = KeyStore.getInstance("PKCS12");
        deviceKeystore.load(new FileInputStream("device.p12"), "devicePass".toCharArray());
        
        KeyManagerFactory kmf = KeyManagerFactory.getInstance(KeyManagerFactory.getDefaultAlgorithm());
        kmf.init(deviceKeystore, "devicePass".toCharArray());
        
        // Load CA certificate for server validation
        KeyStore trustStore = KeyStore.getInstance("PKCS12");
        trustStore.load(new FileInputStream("ca.p12"), "caPass".toCharArray());
        
        TrustManagerFactory tmf = TrustManagerFactory.getInstance(TrustManagerFactory.getDefaultAlgorithm());
        tmf.init(trustStore);
        
        // Create SSL context
        SSLContext sslContext = SSLContext.getInstance("TLSv1.3");
        sslContext.init(kmf.getKeyManagers(), tmf.getTrustManagers(), new java.security.SecureRandom());
        
        return sslContext;
    }
    
    public void connectWithMTLS(String hostname, int port) throws Exception {
        SSLContext sslContext = createMutualTLSContext();
        SSLSocketFactory factory = sslContext.getSocketFactory();
        
        SSLSocket socket = (SSLSocket) factory.createSocket(hostname, port);
        socket.startHandshake();
        
        System.out.println("Mutual TLS handshake completed");
        System.out.println("Server certificate validated via CA");
    }
}

JWT for IoT Device Authentication

JWT TOKEN GENERATION
import io.jsonwebtoken.Jwts;
import io.jsonwebtoken.SignatureAlgorithm;
import java.security.KeyPair;
import java.util.Date;

public class JWTAuthenticator {
    private KeyPair keyPair;
    
    public String generateDeviceToken(String deviceId, int ttlHours) {
        Date now = new Date();
        Date expiration = new Date(now.getTime() + ttlHours * 3600 * 1000);
        
        return Jwts.builder()
            .setSubject(deviceId)
            .setIssuedAt(now)
            .setExpiration(expiration)
            .claim("deviceType", "sensor")
            .claim("capabilities", new String[]{"read", "write"})
            .signWith(keyPair.getPrivate(), SignatureAlgorithm.ES256)
            .compact();
    }
    
    public boolean validateToken(String token) {
        try {
            Jwts.parserBuilder()
                .setSigningKey(keyPair.getPublic())
                .build()
                .parseClaimsJws(token);
            return true;
        } catch (Exception e) {
            System.err.println("Token validation failed: " + e.getMessage());
            return false;
        }
    }
}
EXAMPLE: JWT Structure
Header (Base64):
{"alg":"ES256","typ":"JWT"}

Payload (Base64):
{
  "sub": "device-001",
  "iat": 1708300000,
  "exp": 1708386400,
  "deviceType": "sensor",
  "capabilities": ["read", "write"]
}

Signature:
HMACSHA256(

Encryption & Secure Communication

Protecting Data at Rest and in Transit

Encryption transforms data into unreadable ciphertext that can only be decrypted with the correct key. IoT systems must encrypt data in transit (network communications) and at rest (flash storage, databases). Key management is as important as encryption itself.

ENCRYPTION ALGORITHMS
  • AES-256-GCM: Symmetric, authenticated encryption. Industry standard for data at rest and TLS 1.3.
  • ChaCha20-Poly1305: Symmetric, fast, authenticated. Good for mobile/IoT.
  • ECDH (P-256, X25519): Key exchange. Establishes shared secret without transmitting it.
  • RSA-2048/4096: Asymmetric. Key exchange, digital signatures, certificates.
  • Ed25519: Modern, fast digital signatures.

AES Encryption with Java

AES-256-GCM ENCRYPTION/DECRYPTION
import javax.crypto.*;
import javax.crypto.spec.*;
import java.security.SecureRandom;
import java.util.Base64;

public class AESCrypto {
    private static final int GCM_TAG_LENGTH = 128;
    private static final int IV_LENGTH = 12;
    private static final int KEY_LENGTH = 32; // 256 bits
    
    public byte[] generateKey() {
        byte[] key = new byte[KEY_LENGTH];
        new SecureRandom().nextBytes(key);
        return key;
    }
    
    public byte[] encrypt(byte[] plaintext, byte[] key) throws Exception {
        // Generate IV
        byte[] iv = new byte[IV_LENGTH];
        new SecureRandom().nextBytes(iv);
        
        // Configure cipher
        Cipher cipher = Cipher.getInstance("AES/GCM/NoPadding");
        SecretKeySpec keySpec = new SecretKeySpec(key, "AES");
        GCMParameterSpec gcmSpec = new GCMParameterSpec(GCM_TAG_LENGTH, iv);
        cipher.init(Cipher.ENCRYPT_MODE, keySpec, gcmSpec);
        
        // Encrypt
        byte[] ciphertext = cipher.doFinal(plaintext);
        
        // Return IV + ciphertext (IV required for decryption)
        byte[] result = new byte[IV_LENGTH + ciphertext.length];
        System.arraycopy(iv, 0, result, 0, IV_LENGTH);
        System.arraycopy(ciphertext, 0, result, IV_LENGTH, ciphertext.length);
        
        return result;
    }
    
    public byte[] decrypt(byte[] encryptedData, byte[] key) throws Exception {
        // Extract IV and ciphertext
        byte[] iv = new byte[IV_LENGTH];
        byte[] ciphertext = new byte[encryptedData.length - IV_LENGTH];
        System.arraycopy(encryptedData, 0, iv, 0, IV_LENGTH);
        System.arraycopy(encryptedData, IV_LENGTH, ciphertext, 0, ciphertext.length);
        
        // Configure cipher for decryption
        Cipher cipher = Cipher.getInstance("AES/GCM/NoPadding");
        SecretKeySpec keySpec = new SecretKeySpec(key, "AES");
        GCMParameterSpec gcmSpec = new GCMParameterSpec(GCM_TAG_LENGTH, iv);
        cipher.init(Cipher.DECRYPT_MODE, keySpec, gcmSpec);
        
        // Decrypt and verify authentication tag
        return cipher.doFinal(ciphertext);
    }
}

Key Management

KEY MANAGEMENT PRINCIPLES
  • Never hardcode keys in source code or config files
  • Use hardware security modules (HSM) when available
  • Rotate keys regularly without device re-provisioning
  • Never transmit keys on plaintext channels
  • Encrypt keys at rest with device-unique keys
SECURE KEY STORAGE WITH ANDROID KEYSTORE
import android.security.keystore.*;

public class SecureKeyStorage {
    private static final String KEY_ALIAS = "iot_device_key";
    
    public void generateKey() throws Exception {
        KeyPairGenerator kpg = KeyPairGenerator.getInstance(
            KeyProperties.KEY_ALGORITHM_EC, "AndroidKeyStore"
        );
        
        KeyGenParameterSpec spec = new KeyGenParameterSpec.Builder(
            KEY_ALIAS,
            KeyProperties.PURPOSE_SIGN | KeyProperties.PURPOSE_VERIFY
        )
        .setDigests(KeyProperties.DIGEST_SHA256)
        .setSignatureAlgorithms(KeyProperties.SIGNATURE_ECDSA_SHA256)
        .setUserAuthenticationRequired(false)
        .build();
        
        kpg.initialize(spec);
        kpg.generateKeyPair();
    }
    
    public byte[] signData(byte[] data) throws Exception {
        KeyStore ks = KeyStore.getInstance("AndroidKeyStore");
        ks.load(null);
        PrivateKey privateKey = (PrivateKey) ks.getKey(KEY_ALIAS, null);
        
        Signature signature = Signature.getInstance("SHA256withECDSA");
        signature.initSign(privateKey);
        signature.update(data);
        
        return signature.sign();
    }
}

Secure Boot and Firmware Integrity

FIRMWARE SIGNATURE VERIFICATION
import java.security.Signature;
import java.security.spec.X509EncodedKeySpec;
import java.security.KeyFactory;

public class FirmwareValidator {
    private PublicKey manufacturerKey;
    
    public boolean validateFirmware(byte[] firmware, byte[] signature) 
            throws Exception {
        // Load manufacturer public key
        X509EncodedKeySpec spec = new X509EncodedKeySpec(manufacturerKeyBytes);
        manufacturerKey = KeyFactory.getInstance("EC").generatePublic(spec);
        
        // Verify signature
        Signature sig = Signature.getInstance("SHA256withECDSA");
        sig.initVerify(manufacturerKey);
        sig.update(firmware);
        
        return sig.verify(signature);
    }
    
    // In bootloader firmware
    public void applyUpdate(byte[] firmware, byte[] signature) {
        if (validateFirmware(firmware, signature)) {
            // Write to flash
            writeToFlash(firmware);
            System.out.println("Firmware signature valid, update applied");
        } else {
            System.err.println("Firmware signature invalid, update rejected");
            System.err.println("This could be malicious firmware");
            // Do not apply update
        }
    }
}

IoT Security Best Practices

Guidelines for Building Secure IoT Systems

DEVICE HARDENING
  • Default Credentials: Never ship with admin/admin or factory passwords
  • Change on First Boot: Force users to set unique credentials
  • Disable Unused Interfaces: Close unused ports (telnet, SSH debug)
  • Secure Boot: Only signed firmware runs
  • JTAG Disable: Disable hardware debugging interfaces in production
  • Physical Tamper Detection: Detect and log/sanitize on physical intrusion
NETWORK SECURITY
  • Always Encrypt: TLS 1.3 for all communications
  • Certificate Pinning: Prevent MITM attacks via certificate validation
  • Firewall Rules: Only allow necessary ports and protocols
  • Network Segmentation: Separate IoT network from corporate network
  • VPN for Remote Access: Never expose device management directly to internet
FIRMWARE UPDATES
  • Signed Updates: Only apply cryptographically signed firmware
  • Rollback Protection: Prevent downgrade attacks
  • Staged Rollout: Test updates on subset before full deployment
  • Verification: Verify integrity before and after update
  • Safe Mode: Recovery mode if update fails
DATA PROTECTION
  • Encrypt at Rest: AES encryption for flash storage
  • Encrypt in Transit: TLS, DTLS, or application-level encryption
  • Data Minimization: Only collect necessary data
  • Anonymization: Strip personally identifiable information where possible
  • Secure Deletion: Securely erase data when decommissioning devices
EXAMPLE: Firmware Update Security Flow
1. Device receives update package (firmware + signature)
2. Verify signature using manufacturer public key
   └─ Invalid: Reject update, log incident
3. Verify version > current version (rollback protection)
   └─ Older: Reject update
4. Calculate hash of package, compare to manifest
   └─ Hash mismatch: Reject update
5. Write to non-active partition (dual-bank flash)
6. Update boot flag to new partition
7. Reboot
8. Bootloader verifies signature again
9. System runs from new partition
10. Keep old partition for N rollback attempts

Security Testing

Testing and Auditing IoT Systems

Security cannot be assumed—it must be tested. Penetration testing, static analysis, and runtime monitoring help identify vulnerabilities before attackers do. Test throughout the development lifecycle, not just before release.

SECURITY TESTING TYPES
  • Static Analysis: Code scanners check for vulnerabilities (e.g., hardcoded secrets)
  • Dynamic Analysis: Test running application for runtime vulnerabilities
  • Penetration Testing: Simulated attacks to find exploitable weaknesses
  • Fuzz Testing: Send malformed input to crash/fault devices
  • Network Security Assessment: Scan for open ports, weak encryption

Monitoring and Intrusion Detection

SIMPLE ANOMALY DETECTION
public class IoTSecurityMonitor {
    private Map<String, DeviceProfile> profiles = new ConcurrentHashMap<>();
    
    public void reportMetrics(String deviceId, DeviceMetrics metrics) {
        DeviceProfile profile = profiles.computeIfAbsent(
            deviceId, k -> new DeviceProfile()
        );
        
        // Check for anomalies
        if (profile.isAnomaly(metrics)) {
            Alert alert = new Alert(
                "SUSPICIOUS ACTIVITY",
                "Anomaly detected for device " + deviceId,
                AlertSeverity.HIGH
            );
            alertManager.sendAlert(alert);
        }
        
        profile.record(metrics);
    }
}

class DeviceProfile {
    private double avgDataRate;
    private double stdDevDataRate;
    private Queue<DeviceMetrics> history = new LinkedList<>();
    
    public boolean isAnomaly(DeviceMetrics metrics) {
        double currentRate = metrics.getDataRate();
        double zScore = (currentRate - avgDataRate) / stdDevDataRate;
        
        // Flag if > 3 standard deviations from mean
        return Math.abs(zScore) > 3;
    }
}
