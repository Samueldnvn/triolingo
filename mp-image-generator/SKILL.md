# Memory Palace Image Generator

## Core Principles

**Every Element Must Be Justified** - No random elements. Each object, character, action, or detail in the image must relate to the concept being memorized through:
- Sound/Phonetic similarity (homophones, near-homophones)
- Obvious symbolic representation
- Action that demonstrates the concept

**Action-Packed** - Images should involve dynamic actions, explosions, transformations, or intense scenes. Static images are less memorable.

**Sound-Based Associations** - Prioritize wordplay and phonetic links over arbitrary visual metaphors:
- "new" → "brand new", "newborn"
- "null" → "nun", "zero"
- "array" → "ray of light"
- "length" → "lens", "long"
- "static" → "statue", "still"

**Multi-Sensory** - Include sounds, textures, motions, and sensations to make the image stick.

**Density Over Celebrity** - Characters only when they serve a sound association. Don't force celebrities or public figures for their own sake.

## Quick Start

1. **Read the CSV** - Identify the definition/third column containing the concept to memorize
2. **Analyze Key Terms** - Break down the definition into important technical terms and keywords
3. **Find Sound Associations** - For each key term, find homophones, near-homophones, or sound-alike words
4. **Construct Narrative** - Create an action-packed scene combining these sound associations
5. **Write Concisely** - Keep dense, short, and impactful

## Sound Association Techniques

### Homophones (Perfect Sound Matches)
- "array" ↔ "ray", "rays"
- "new" ↔ "new", "knew"
- "class" ↔ "glass", "classroom"
- "void" ↔ "void", "avoid"
- "null" ↔ "nun", "null"

### Near-Homophones (Sound-Similar)
- "dynamic" ↔ "dynamite", "die a mic"
- "static" ↔ "statue", "sticks"
- "constructor" ↔ "constriction", "construction"
- "generic" ↔ "genetic", "jen-eric"
- "extends" ↔ "extensive", "extent"

### Word Breakdown & Reassembly
- "parse" → "par" + "see" → "pair of eyes seeing"
- "scanner" → "scan" + "ner" → "scanning nerve
- "substring" → "sub" + "string" → "submarine with strings

### Symbolic Associations (When Sound Fails)
- "binary" → two, dual, twins
- "heap" → pile, mountain, stack
- "queue" → lining up, waiting
- "stack" → pancakes, plates piled

## Image Construction Process

### Step 1: Identify Key Terms
From the definition, extract 3-5 core terms that MUST be represented:
- Data types (String, int, double)
- Methods or operations (.length, .add, .remove)
- Keywords (new, static, public, private)
- Data structures (array, list, tree, graph)

### Step 2: Map Each Term to Sound
Create a mapping table:
```
"String" ↔ "strings/rope"
"array" ↔ "rays of light"
"initialize" ↔ "initiate/start ceremony"
"null" ↔ "nun/blank"
"length" ↔ "lens"
```

### Step 3: Build Action Scene
Construct a dynamic narrative combining all mapped elements:
- **Who/What**: The primary actors/objects
- **Action**: What's happening (exploding, transforming, fighting, constructing)
- **Setting**: Where it takes place
- **Interaction**: How the elements interact with each other

### Step 4: Review for Density
Check that:
- [ ] Every element has a sound-based justification
- [ ] No filler words or unnecessary descriptors
- [ ] The scene is action-packed
- [ ] Short and dense (one compact sentence ideal)

## Examples

### Example 1: "String[] = new String[5]" (new method with null values)

**Key terms:** String, array, new, null

**Sound mapping:**
- String → "strings/rope"
- array → "rays of light"
- new → "brand new"
- null → "nun"

**Result:** "Five brand new nuns standing in a row, eyes closed in prayer, completely empty-headed like blank nulls."

**Why it works:**
- "Five" → array size [5]
- "Brand new" → new keyword
- "Nuns" → null (sound)
- "Empty-headed" → null values

### Example 2: "ArrayList" (dynamic arrays)

**Key terms:** array, list, dynamic

**Sound mapping:**
- array → "rays"
- list → "list"
- dynamic → "dynamite"

**Result:** "Rays of light striking a dynamite list, the list exploding and reforming instantly with new items."

**Why it works:**
- "Rays" → array (sound)
- "Dynamite" → dynamic (sound)
- "Exploding reforming" → shows dynamic nature

### Example 3: ".length" property

**Key terms:** length

**Sound mapping:**
- length → "lens"

**Result:** "A magnifying lens stretching across the entire football field, measuring the distance from goal line to goal line."

**Why it works:**
- "Lens" → length (sound)
- "Stretching" → shows measurement concept
- "Football field" → visualizes length/distance

### Example 4: For-Each Loop

**Key terms:** for, each, enhanced, cannot modify

**Sound mapping:**
- for → "four"
- each → "reach"
- enhanced → "hen"
- cannot modify → "impossible to change"

**Result:** "Four identical birds reaching over a fence, each bird touching the same row array, unable to modify anything."

**Why it works:**
- "Four" → for (sound)
- "Reaching/Each" → each (sound)
- "Birds/Hens" → Enhanced (sound)
- "Unable to modify" → defines the constraint

## CSV Format Expectations

**Standard Format:**
```
,Mnemonic Image,Location,Definition
```

**Column 1:** Filled or empty mnemonic images
**Column 3:** Definitions/concepts to memorize

**Processing:**
- Read all rows
- Identify empty Column 1 cells
- Generate images based on Column 3 definitions
- Maintain CSV structure when writing output

## Quality Checklist

Before finalizing images, verify:

- [ ] **Every element justified** - Can you explain WHY each noun/verb is there based on sound?
- [ ] **No celebrity crutches** - Only use characters if they serve a sound association
- [ ] **Action-packed** - Is there movement, change, transformation?
- [ ] **Sound-based** - Primary mechanism is phonetic/wordplay, not just visual
- [ ] **Dense and concise** - No wasted words, maximum information per character
- [ ] **Multi-sensory** - Includes sounds, textures, motions where possible
- [ ] **Technically accurate** - The image still represents the concept correctly
